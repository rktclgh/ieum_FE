# 프로필 이미지 업로드 실패 안내 설계

## 문제

프로필 크롭 후 공통 S3 업로드가 실패하면 `EditProfileForm`이 예외를 삼키고 사용자에게 아무 안내도 하지 않는다. `useProfileImageUpload`는 성공할 때만 `['me']` 캐시를 갱신하므로 기존 아바타는 보존되지만, 사용자는 실패와 재시도 필요성을 알 수 없다.

모임 생성 화면은 같은 공통 업로드 실패를 이미 `t.imageUploadFailed` 설명으로 노출하고 생성 요청을 중단한다. 이 흐름은 변경하지 않는다.

## 결정

프로필 편집 폼에 로컬 `profileImageUploadError` 상태를 둔다. 새 파일 선택 또는 새 크롭 시작 시 오류를 비우고, `upload(blob)` 거절 시 기존 번역 키 `messages.profileImage.uploadFailed`를 상태에 기록한다. 아바타 영역 아래에 접근 가능한 `Explanation` 오류를 렌더한다.

`ProfileImageEditor`의 닫힘/재시도 흐름은 변경하지 않는다. 업로드 실패 시 기존 `['me']` 캐시와 현재 프로필 이미지를 유지하고, 사용자는 다시 사진을 선택해 재시도할 수 있다.

## 범위

- 프로필 이미지 업로드 실패를 사용자에게 명시한다.
- 기존 번역 키만 사용하며 새 번역 키는 만들지 않는다.
- 정적 소스 계약 테스트로 파일 선택/크롭 재시도 시 오류 초기화, 실패 메시지 매핑, alert 렌더링을 고정한다.
- 모임 생성의 기존 실패 안내가 유지되는지 같은 계약 테스트로 확인한다.

## 비범위

- S3 PUT 구현 변경(백엔드 #150의 범위)
- 모임 생성 화면의 오류 UX 재작성
- 프로필 삭제 실패 UX 변경
- 실패 시 이미지 편집 모달을 열린 상태로 유지하는 UX 재설계
- 계정 생성 시 선택 이미지 업로드 실패 정책 변경

## 불변 조건

1. 업로드 실패는 `['me']` 캐시나 현재 아바타 URL을 변경하지 않는다.
2. 성공 시에는 기존 mutation의 `onSuccess`가 새 profileImageUrl로 캐시를 갱신한다.
3. 오류는 새 파일 선택 또는 다음 크롭 시 사라진다.
4. 모임 생성은 업로드 실패 뒤 모임 생성 API를 호출하지 않는다.

## 검증

- 기존 Node 정적 계약 테스트에 수정 전 실패하는 항목을 추가한다.
- 테스트는 프로필 폼이 오류 상태를 초기화하고, catch에서 기존 번역 키를 기록하며, `role="alert"` 오류 설명을 렌더하는지 확인한다.
- 모임 생성의 기존 `t.imageUploadFailed` catch/early return도 계약으로 확인한다.
- `pnpm test:contracts`, `pnpm lint`, `pnpm typecheck`를 실행한다.
